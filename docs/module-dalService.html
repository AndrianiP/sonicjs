Module: dalService

DAL (Data Access Layer) Service module The dalService should rarely be used directly. Typically you will want to use the data service that accepts a session ID. It is ok to use the dal service when the system needs to perform a data operation that is not tied to a user action. Examples include: Checking for content while the system is starting up, writing logs or analytics, etc. Note that you are bypassing any and all permissions settings when using the dal service directly.
